# PRODUCTION SYSTEM ARCHITECTURE

## Authentication Flow

```
User Login (bharatjadhav1971@gmail.com)
    ↓
Supabase Auth validates credentials
    ↓
Session token stored in cookies (sb-auth-token)
    ↓
Middleware verifies token on each request
    ↓
Dashboard loads with user's data
    ↓
Real-time subscriptions activated
```

## Real-Time Synchronization Architecture

### Push-Based Automatic Sync

```
Device A (Laptop)
    ↓
User adds item
    ↓
Item saved to Supabase
    ↓
Supabase trigger fires
    ↓
Real-time subscription notifies all connected clients
    ↓
Device B (Mobile) receives update automatically
    ↓
Dashboard refreshes without user action
```

```

```

## Security

### Row Level Security (RLS) Policies Applied

✓ **Items Table**: User can only view/edit/delete their own items
✓ **Categories Table**: User can only view/edit/delete their own categories
✓ **Units Table**: User can only view/edit/delete their own units
✓ **Price Tiers Table**: Access controlled through item ownership
✓ **Users Table**: User can only view their own profile

### Authentication Security

✓ Supabase handles password hashing (bcrypt)
✓ JWT tokens with automatic expiration
✓ HTTP-only cookies for token storage
✓ Middleware validates every request
✓ Session refresh automatic

## Multi-Device Synchronization

### How It Works

1. **Device A makes changes** → Data saved to Supabase
2. **Database triggers real-time event** → Notifies all subscribers
3. **Device B receives notification** → Automatic UI update
4. **No manual refresh required** → Full automatic sync

## API Endpoints Used

- `POST /auth/v1/token` - Supabase session management
- `GET /rest/v1/items` - Fetch user's items (with RLS)
- `POST /rest/v1/items` - Create items (with RLS)
- `PATCH /rest/v1/items/:id` - Update items (with RLS)
- `DELETE /rest/v1/items/:id` - Delete items (with RLS)
- `realtime:*` - Real-time subscriptions
